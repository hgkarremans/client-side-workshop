Over deze applicatie

Deze applicatie is gemaakt in het kader van het vak 'Clientside web frameworks' van Avans Hogeschool Breda.

Ticketshop

In mijn project ga ik een applicatie ontwerpen waarbij gebruikers tickets kunnen reserveren om naar wedstrijden van een voetbalclub te kunnen gaan. Voetbal clubs hebben en divisie en voetbal spelers. De gebruikers kunnen vriendschappen hebben met andere gebruikers. Een vriendschap heeft een gebruiker 1 en gebruiker 2, deze combi maakt hem uniek. Gebruikers hebben een Id, voornaam, achternaam, email, telefoonnummer, wachtwoord, adres, geslacht en vervoer. Tickets hebben een Id, prijs, datum en status. Voetbalclubs hebben een Id, naam, oprichtingsDatum en stadion. Divisies hebben een Id, naam en een klasse. Voetballers hebben een Id, naam en een rugnummer.

Userstories

Voor deze applicaties zijn enkele userstories opgesteld. Deze zijn hieronder te lezen.

Nr Userstory Toelichting
1 Als gebruiker wil ik een overzicht van tickets kunnen zien, zodat ik kan zien welke tickets er beschikbaar zijn. Overzicht is een grid waarin de beschibare tickets te zien zijn.
2 Als admin wil ik een ticket kunnen toevoegen, zodat er altijd tickets beschikbaar zijn voor de gebruikers. Er is een create knop waarmee admins tickets kunnen toevoegen, deze knop is alleen beschikbaar als de gebruiker Een admin role heeft. Als dit niet is, is de knop onzichtbaar voor de gebruiker. Hierna is een schema te zien waarin de benodigde data kan worden ingevuld.
3 Als admin wil ik tickets kunnen aanpassen en verwijderen, zodat ze altijd up to date zijn. Sin het een edit en delete knop beschikbaar als een gebruiker de benodigde admin role heeft. Als dit zo is kan via edit een schema worden veranderd om het ticket aan te passen en via delete kan het ticket worden verwijderd.
4

Als gebruiker wil ik een ticket kunnen claimen, zodat het ticket alleen van mij is.

Op de detail page van de tickets is er een claim button, waarop een gebruiker kan klikken als het ticket nog niet geclaimd is door een andere gebruiker. Als dit wel zo is, is de button niet beschikbaar.
5

Als gebruiker wil ik een overzicht van de spelers van de club kunnen zien, zodat ik weet welke spelers er spelen.

Op de ticketpage is een overzicht te zien van de spelers die spelen bij de geselecteerde clubs die bij de tickets horen.
7

Als gebruiker wil ik een vriendschap kunnen aangaan met een andere gebruiker.

Scherm om vriendschappen te kunnen beginnen via een email input veld. Als de user bestaat, wordt de vriendschap aangegaan.
6

Als geruiker wil ik een vriendschap kunnen inzien en verwijderen, zodat ik kan zien met wie ik een vriendschap heb.

Scherm om spelers te verwijderen en in te kunnen zien met welke gebruiker de gebruiker vrienden is. Als de gebruiker geen vrienden heeft, kan hij een nieuwe vriendschap aangaan. Een gebruiker kan geen vriendschap met zichzelf aangaan. En een vriendschap is uniek, dus een gebruiker kan niet 2 keer vrienden zijn met dezelfde gebruiker.
